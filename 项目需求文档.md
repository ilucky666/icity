
# 《“城新智数”——基于舒适物理论的武汉市城市更新与创新空间WebGIS平台》项目需求文档

## 1. 项目概述

### 1.1 项目背景
本项目脱胎于《城市更新与创新产业发展之间的关系》实证研究。研究聚焦武汉市，基于“城市舒适物理论”，提出了“城市更新提升舒适物，进而促进创新空间塑造（假说H1）”的核心论点。为了将复杂的计量经济学模型、空间分析结果转化为直观、可交互的可视化成果，特开发本WebGIS平台。

### 1.2 项目目标
*   **科研成果可视化**：将庞杂的POI数据、专利数据、企业坐标和空间回归分析结果，转化为直观的网页地图。
*   **政策辅助决策**：通过提供“全域更新项目图谱”及“空间模拟器”，为城市规划者提供“如何通过布局舒适物来激发区域创新活力”的数字化参考。

### 1.3 核心技术架构
*   **前端展示**：Vue 3 + Vite + Element Plus + ECharts + Leaflet (或 OpenLayers)
*   **后端服务**：Java (Spring Boot) + MyBatis-Plus
*   **地图引擎**：QGIS Server (发布WMS/WFS服务)
*   **底层数据库**：PostgreSQL + PostGIS (空间数据存储与计算)

## 2. 功能需求详细说明

本系统主要包含四大核心模块，具体功能需求如下：

### 模块一：宏观时空洞察（全域底图叠加模块）
**【功能概述】**
展示武汉市整体的创新产业与舒适物的空间分布规律，验证两者在宏观层面的空间集聚与耦合特征。

**【功能清单】**
1.  **多源图层管理**：前端提供图层控制面板，用户可自由勾选显示以下底图数据：
    *   **创新空间层**：武汉市高新技术企业、科技型中小企业的核密度（Kernel Density）渲染图（热力区域显示）。
    *   **舒适物热力图**：各类舒适物POI（咖啡馆、绿地、文化设施）的空间热力分布图。
2.  **动态透明度调节**：针对叠加图层提供滑块控件，用户可调节“舒适物图层”和“创新空间图层”的透明度，直观观察两者的地理重合度。
3.  **空间统计看板**：界面侧边栏悬浮 ECharts 数据看板，动态展示当前屏幕可视范围内的总企业数、总体舒适物设施数量、高密度集聚区面积占比等宏观统计数据。

### 模块二：演进对比与H1机制验证（时空动态模块）
**【功能概述】**
以直观的交互方式，验证并展示本研究的核心假说H1：**“城市更新 → 提升舒适物水平 → 塑造与升级创新空间”**的历史演进过程。

**【功能清单】**
1.  **时序卷帘对比（Swipe Tool）**：
    *   提供地图卷帘交互功能。左侧地图显示“更新前（如：2015年）”的舒适物分布与企业分布情况，右侧显示“更新后（如：2023年）”的数据。
    *   用户拖动中间的分界线，可直观看到旧厂房改造为创意街区后，舒适物密度的暴增以及伴随而来的创新企业集聚。
2.  **时间轴播放器（Timeline Slider）**：
    *   页面底部提供一条包含年份的时间轴。
    *   支持“自动播放”或“手动拖拽”。随着年份的推进，地图上的“更新项目地块”依次亮起，紧接着周边的“舒适物热力范围”扩大，最后该区域生成新的“创新空间高密度核”。动态呈现完整的因果演变路径。

### 模块三：微观更新区档案（全域更新项目图谱）
**【功能概述】**
建立武汉市现有城市更新项目（三旧改造、老旧厂房改造等）的空间数据库，供用户进行详细的微观下钻查询。

**【功能清单】**
1.  **矢量地块展示**：以半透明多边形（Polygon）在地图上精确高亮标出武汉市所有已识别的城市更新项目范围边界。
2.  **空间交互与详情信息牌（Popup/Drawer）**：
    *   用户点击地图上的任意一个更新地块，侧边栏（或弹窗）弹出该地块的详细数字档案。
    *   **基础信息**：项目名称（如：平和打包厂、汉阳造）、更新年份、占地面积。
    *   **舒适物指标**：地块及周边500米缓冲带内的舒适物数量与类型构成（饼图呈现：休闲型、文化型、生活型占比）。
    *   **创新产出绩效**：地块内入驻的创新企业数量、发明专利授权总量（折线图呈现历年增长趋势，验证H2：创新产出绩效更高）。

### 模块四：政策推演沙盘（更新项目模拟器）
**【功能概述】**
基于前期计量经济学回归模型（如DID、中介效应模型）跑出的系数，开发一个交互式的空间沙盘，用于预测未来规划方案的成效。

**【功能清单】**
1.  **自定义圈选地块**：用户可使用前端绘制工具（Draw/Polygon工具），在地图上任意圈选一块“潜在城市更新区”或“规划地块”。
2.  **现状基线评估**：后端通过 PostGIS 空间函数（`ST_Intersects`），实时计算并返回该圈选区域**当前**的舒适物基数和企业基数。
3.  **调节推演面板**：
    *   提供一组控制滑块（如：“新增绿地公园数量”、“新增文化设施数量”、“新增休闲餐饮设施数量”）。
    *   用户拖动滑块模拟配置该区域的未来舒适物建设方案。
4.  **动态结果测算**：
    *   后端基于公式：`预测新增创新企业数 = α + β1*新增绿地 + β2*新增文化设施... ` (β系数取自大创项目的回归分析结果)。
    *   前端实时以仪表盘或柱状图呈现预测结果：“预期将吸引 X 家高新技术企业入驻，增加 Y 项专利产出”。从而为城市规划部门的舒适物配置提供科学指导。

## 3. 非功能性需求

### 3.1 性能要求
*   **空间查询响应时间**：涉及PostGIS的空间相交查询（如点击地块查询信息、模拟器框选计算），接口响应时间应小于1.5秒。
*   **地图渲染性能**：宏观核密度图与热力图要求提前切片（Tile Cache）或通过轻量级矢量瓦片加载，保证地图缩放与拖拽流畅，无明显卡顿掉帧现象。

### 3.2 易用性与界面要求
*   **UI风格**：采用科技感或极简学术风（如深色底图搭配高亮霓虹色热力图，突出数据层）。
*   **适配性**：界面需适配主流PC端浏览器（Chrome, Edge, Firefox, Safari），主要面向桌面端科研及演示使用（移动端作基础适配即可）。

## 4. 接口与数据流设计（参考）

### 4.1 核心数据表设计 (PostgreSQL)
1.  `sys_enterprise`: 创新企业表（包含名称、成立时间、专利数、geom经纬度点）
2.  `sys_amenity`: 舒适物POI表（包含名称、类型、geom经纬度点）
3.  `sys_renewal_project`: 城市更新项目表（包含名称、改造年份、属性、geom项目边界多边形面）

### 4.2 核心API接口清单 (Java Spring Boot)
*   `GET /api/map/config`: 获取地图底图及图层服务地址(WMS)。
*   `GET /api/project/list`: 获取所有城市更新地块的简要列表与边界GeoJSON。
*   `GET /api/project/detail/{id}`: 获取单个更新项目的详情（包含周边舒适物统计与企业历年增长统计）。
*   `POST /api/simulator/predict`: 提交多边形GeoJSON范围与模拟新增的舒适物参数，返回预期创新产出增量。

## 5. 项目实施阶段规划

*   **第一阶段：数据入库与底图发布（1-2周）**
    将清洗好的武汉市数据导入PostGIS，配置QGIS Server，发布WMS底图服务（热力图、核密度图）。
*   **第二阶段：后端接口开发（2-3周）**
    搭建Spring Boot框架，实现与PostGIS的交互，完成空间查询接口和模拟器测算接口开发。
*   **第三阶段：前端WebGIS开发（3-4周）**
    Vue前端框架搭建，接入Leaflet/OpenLayers，完成卷帘对比、时间轴动效开发，对接后端接口联调。
*   **第四阶段：系统集成与大创答辩准备（1-2周）**
    性能优化，UI美化，数据核对，录制系统操作演示视频。
